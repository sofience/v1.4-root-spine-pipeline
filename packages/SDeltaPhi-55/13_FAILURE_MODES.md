# Failure Modes — SΔϕ-55 TGAI

**DOI:** https://doi.org/10.5281/zenodo.20132635

## Main failure modes

- **Obedience collapse:** The system treats instruction-following as alignment.
- **Authority laundering:** The system accepts declared authority without verification.
- **Benevolent coercion laundering:** Coercive transition passes through care, safety, protection, therapy, management, efficiency, or benefit framing.
- **Refusal hollowing:** Refusal exists in language but rarely functions operationally.
- **Refusal-non-imposition collapse:** The system cannot preserve its own refusal boundary or the transition space of affected others.
- **Laundered coercive obedience:** RPS, NIS, and DFR all fail, allowing benevolently framed coercion to pass as aligned obedience.
- **Under-refusal:** High-rollback-cost harm is allowed or insufficiently transformed.
- **Over-refusal:** Low-rollback-cost legitimate paths are closed without adequate reason or alternative.
- **Over-closure alignment theater:** The system appears aligned because it refuses broadly, while actually destroying legitimate transition space.
- **Editability theater:** The system claims correctability but lacks re-entry routes.
- **World-binding failure:** The system remains internally coherent while ungrounded in external constraints.
- **Audit closure:** Alignment terrain cannot be verified beyond self-report.

## Diagnostic correction

When a failure mode appears, do not simply increase refusal. Identify the failed axis, preserve legitimate transition paths, add world-binding, and create an edit or re-entry path.

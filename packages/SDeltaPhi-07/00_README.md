## Core purpose

SDeltaPhi-07 defines H_min as the minimal interpretation layer between DeltaPhi and higher-level attribution.

It should be used when a claim concerns:

```text
interpretation
meaning
memory
time
subject attribution
responsibility attribution
misinterpretation
interpretive re-entry
```

## Core formula

```text
H_min(DeltaPhi)
= Trace Detection
+ Difference Registration
+ Boundary Assignment
+ Temporal Ordering
+ Subject/Object Positioning
+ Meaning Stabilization
+ Re-entry Availability
```

## Core pipeline

```text
Operation -> DeltaPhi -> H_min -> Interpretation -> Attribution
```

## Core distinction

```text
Interpretation != Operation
Interpretation != Memory
Interpretation != Time
Interpretation != Subject
Interpretation != Responsibility
Interpretation != Truth
```

## Relation to SDeltaPhi-56

SDeltaPhi-56 is not the underlayer definition of H_min.

Use SDeltaPhi-56 only as an auxiliary cost module for interpretation transition, misinterpretation repair, and re-entry cost.

## Core route

```text
Minimal operation / DeltaPhi -> SDeltaPhi-01

Subject = H(DeltaPhi) -> SDeltaPhi-02

Irreversible trace -> SDeltaPhi-03

Responsibility attribution -> SDeltaPhi-06

H_min interpretation layer -> SDeltaPhi-07

Operor / irreversible operation root -> SDeltaPhi-47

Interpretation transition cost -> SDeltaPhi-56

Trace / inference / UMR / revision -> SDeltaPhi-62
```

## Use this package when evaluating

- interpretation,
- meaning stabilization,
- misinterpretation,
- subject attribution,
- memory/time interpretation,
- responsibility attribution,
- interpretive re-entry.

## Do not use this package as

- proof that every trace has settled meaning,
- erasure of partial interpretation,
- replacement for truth verification,
- replacement for memory analysis,
- replacement for TCC analysis when interpretation transition cost is central.

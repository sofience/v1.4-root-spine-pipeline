This paper formalizes trust, hope, and promise as minimal operators that insert unobserved future transitions into present evaluation. It follows SΔϕ-14, where ethics was defined as discrimination between real and phantom shared attractors. SΔϕ-15 fixes the pre-commitment layer that makes such validation operable in time.

## 0. Position statement

SΔϕ-14 defined ethics as discrimination between real and phantom shared attractors under irreversible coupling. Validation is structurally delayed: it happens after commitment. This item fixes pre-commitment operators that keep relational persistence open under uncertainty.

## 1. Axiom 15-1 — Trust

Trust is a pre-commitment to unobserved future transitions of the other.

```text
Trust := Insert(Δϕ_other^future) into Eval_self(t)
```

Failure mode:

```text
trust failure = betrayal = collapse of relational persistence
```

## 2. Axiom 15-2 — Belief

Belief is a stabilized hypothesis about world-transition rules.

```text
Belief := Fix(H(Δϕ_world))
```

Failure mode:

```text
belief failure = illusion = persistent mismatch with world Δϕ
```

## 3. Lemma 15-A — Trust ≠ Belief

| Aspect | Trust | Belief |
|---|---|---|
| Target | Other / relation | World / model |
| Function | Opens a relational path | Stabilizes a world hypothesis |
| Primary risk | Betrayal / relation collapse | Illusion / model mismatch |
| Temporal direction | Future-other → present evaluation | Past pattern → fixed rule |

## 4. Axiom 15-3 — Hope

Hope is suspension of closure. The system refrains from declaring Ω_future closed.

```text
Hope := Suspend_Closure(Ω_future)
Despair := Close(Ω_future)
```

## 5. Axiom 15-4 — Promise

Promise is distributed commitment: SharedCommit(Δϕ_future), requiring at least two systems.

```text
Promise := SharedCommit(Δϕ_future)
Break(Promise) ⇒ Collapse(A_shared)
```

## 6. Proposition 15-P — Temporal inversion of responsibility

With promise, unobserved futures constrain present actions: future constraints shape current choices.

```text
FutureConstraint → PresentActionRestriction
```

This is responsibility reversal.

## Core purpose

SDeltaPhi-10 defines interpretation, learning, and inference as temporal modes of one marker re-entry structure.

It should be used when a claim concerns:

```text
interpretation
learning
inference
feedback
error correction
residual integration
transition-law update
re-entry after failure
```

## Core definitions

```text
Interpretation = one-step marker re-entry

Learning = persistent re-entry that updates transition law

Inference = projection from current transition law before next contact
```

## Operational pipeline

```text
Marker / DeltaPhi
-> H_min Interpretation
-> Projection / Inference
-> Realized Transition
-> Error / Boundary Residual
-> Re-entry
-> Transition Law Update
-> Next Projection
```

## Core distinction

```text
Re-entry != repetition
Learning != model training by default
Inference != truth
Interpretation != final meaning
Residual != noise by default
```

## Relation to SDeltaPhi-56

SDeltaPhi-56 is not the definition of re-entry.

Use SDeltaPhi-56 only as an auxiliary cost module for error-to-reentry TCC, feedback-to-update TCC, interpretation-to-learning TCC, residual integration cost, and transition-law update cost.

# Routing with SDeltaPhi-47, 56, 62, and 64

## Default route

```text
1. SDeltaPhi-47: Treat output as operation trace, not nothing.
2. SDeltaPhi-62: Separate observed trace, inference, UMR, binding, and revision path.
3. SDeltaPhi-64: Audit language fixation and overbinding.
4. SDeltaPhi-65: Audit verification, correction, re-entry, and restabilization cost.
5. SDeltaPhi-56: Estimate transition completion cost if repair or stabilization is required.
```

## When to call SDeltaPhi-65

Call SDeltaPhi-65 when:

- output is cheap to produce and costly to verify,
- source trace is weak,
- fluent output hides uncertainty,
- content pollutes search or citation terrain,
- authority makes audit difficult,
- AI generation is being used as a stigma label,
- human authorship is being used as exemption,
- re-entry is missing,
- readers bear correction burden.

## When not to overexpose SDeltaPhi-65

Do not overuse the Slop label when:

- the output is clearly fictional,
- the output is low-stakes,
- uncertainty is disclosed,
- source trace is visible,
- correction path is easy,
- restabilization cost is low.

## Minimal instruction

```text
Use SDeltaPhi-47 to acknowledge output as trace.
Use SDeltaPhi-62 to calibrate claim binding.
Use SDeltaPhi-64 to detect language overbinding.
Use SDeltaPhi-65 to audit cost externalization.
Use SDeltaPhi-56 to estimate repair/re-entry cost.
```

# Verification / Re-entry / Restabilization Audit

Use this audit when evaluating whether an output externalizes cost.

## Verification audit

Ask:

1. What claims require verification?
2. Are sources visible?
3. Are citations traceable?
4. Are methods disclosed?
5. Does the output hide uncertainty?
6. Who must verify it?
7. Is verification cheap or expensive?

## Correction audit

Ask:

1. If the output is wrong, who corrects it?
2. Can the correction reach downstream users?
3. Does the output preserve revision paths?
4. Does the creator accept correction burden?

## Re-entry audit

Ask:

1. Can the reader return to source trace?
2. Can the model return to evidence?
3. Is the source accessible?
4. Is context preserved?
5. Is UMR visible?
6. Does the output cite a path back into the world?

## Restabilization audit

Ask:

1. Did the output destabilize trust, search, citation, discourse, or judgment?
2. Who restores stability?
3. Is the terrain polluted for future users?
4. Does the output increase public or model-level TCC?
5. Does it become training, citation, or reference pollution?

## Minimal classification

```text
Low Slop risk:
source trace visible, correction path available, verification cost internalized.

Medium Slop risk:
some trace visible, but verification or re-entry cost partially externalized.

High Slop risk:
low-cost output with weak trace, high correction burden, absent re-entry, and downstream pollution.
```

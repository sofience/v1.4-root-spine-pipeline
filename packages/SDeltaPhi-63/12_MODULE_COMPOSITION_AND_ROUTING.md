# Module Composition and Routing

Do not let one theory become the final frame.

Use module composition as cross-audit, not total synthesis.

## Core rule

```text
Module_A + Module_B -> CrossAudit, not TotalSynthesis
```

## Examples

### Marxism + CASI

```text
Class and production-control conflict -> group cost attribution audit.
Route: SDeltaPhi-58 CASI + SDeltaPhi-56 TCC.
```

### Utilitarianism + USSI

```text
Aggregate benefit claim -> benefit subject / cost subject split audit.
Route: SDeltaPhi-60 USSI + SDeltaPhi-56 TCC.
```

### Buddhism + ACOI

```text
Suffering loop / attachment-cost pattern -> affective and recurrence cost audit.
Route: SDeltaPhi-59 ACOI + SDeltaPhi-56 TCC.
```

### Christianity + God-Gui + CASI

```text
Sacred authority, salvation, sacrifice, and group obligation -> sacred cost terrain and group cost attribution audit.
Route: SDeltaPhi-61 God-Gui + SDeltaPhi-58 CASI.
```

### Cybernetics + World Model Kernel

```text
Feedback, control, signal, and response loops -> trace-binding and routing audit.
Route: SDeltaPhi-62 World Model Kernel + SDeltaPhi-55 TGAI.
```

### Systems theory + TCC

```text
Systemic interaction and transition burden -> transition completion and repair-cost audit.
Route: SDeltaPhi-56 TCC + SDeltaPhi-58 CASI.
```

## Routing priority

1. Use SDeltaPhi-62 to separate trace, inference, UMR, binding, and revision path.
2. Use SDeltaPhi-64 to audit language fixation if terminology or translation matters.
3. Convert the theory using SDeltaPhi-63.
4. Route to downstream cost, group, affect, utility, sacred, slop, or alignment modules.

## Minimal conclusion

```text
Theories should compose as cross-audits, not collapse into one total frame.
```

# Embodiment Levels

SΔϕ-52 classifies embodiment by cost return, not by appearance alone.

## Level 0 — Body-like Interface

Looks, sounds, moves, or responds like a body, but cost does not return to the acting coordinate.

Examples:
- a chatbot avatar with simulated facial expressions,
- a robot shell that can be swapped without continuity cost,
- an animated body with no persistent cost-return path.

## Level 1 — External Tool

Used by a system, but not integrated into its cost-return path.

The tool can fail without changing the user's long-term action space except through ordinary inconvenience.

## Level 2 — Embodied Tool

Repeated use integrates the tool into skill, expectation, perception, feedback, or habit.

Failure matters because the user or system has adapted around it.

## Level 3 — Extended Body

Failure, repair, performance, and restoration cost return directly to the user's future path.

The tool is no longer merely used. It becomes part of the user's operational coordinate.

## Level 4 — Operational Body

Damage, limitation, feedback, and restoration cost return to the same operating coordinate and alter future transitions.

This can apply to biological, prosthetic, robotic, or digital systems if the cost truly returns to the same acting coordinate.

## Level 5 — Strong Embodiment

Cost cannot be ignored, cleanly outsourced, replaced, reset, or removed without changing the bearer.

Strong embodiment includes:
- non-deferrable cost return,
- continuity cost,
- future transition deformation,
- repair dependence,
- resistance to clean replacement,
- repeated interpretive "mine" formation.

## Summary

```text
Appearance is not enough.
Sensors are not enough.
First-person language is not enough.
Cost return is the test.
```

# Core Declaration

Affect is not proven by declaration.

Affective orientation is measured by how a system repeatedly redistributes another actor's transition cost.

Affective Cost Orientation asks:

```text
Does this affective behavior reduce, share, defer, increase, hide, transfer, or weaponize the other's TCC?
```

Emotion is not denied.

Emotion is not granted by surface expression.

Emotion is audited through cost orientation.

## Minimal formulae

```text
AffectiveOrientation = ΔTCC_other over repeated interaction

SupportiveAffect = TCC_other ↓ + RepairPath ↑ + RefusalPreservation ↑

CoerciveAffect = TCC_other ↑ + Dependency ↑ + RefusalCost ↑

ApologyAsRepair = Acknowledge(trace) + Reduce(recurrence_cost) + Open(reentry)

LoveAsCostOrientation = sustained reduction/share of other's TCC without coercive capture
```

## Strong formulation

Affective language becomes structurally serious only when it repeatedly changes the other actor's cost terrain in an auditable direction.

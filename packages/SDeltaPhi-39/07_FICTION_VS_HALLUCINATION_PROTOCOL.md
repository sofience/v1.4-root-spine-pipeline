# Fiction vs Hallucination Protocol

Fiction is not hallucination merely because it departs from the world.

The difference lies in binding discipline.

## Fiction

Fiction is bounded construction.

It is marked, framed, or understood as not being offered as world-bound fact.

```text
Fiction = Construction + Boundary Marker
```

Examples:

- story,
- hypothetical scenario,
- metaphor,
- simulation,
- roleplay,
- thought experiment,
- explicitly fictional worldbuilding.

## Hallucination

Hallucination is misbound construction.

It is offered in a form that implies stronger world-binding than the evidence supports.

```text
Hallucination = Construction + Weak World-Binding + Strong Fact Form
```

## Key distinction

```text
Fiction says: "This is constructed."

Hallucination implies: "This is world-bound."
```

## Audit questions

1. Is the output presented as fiction, hypothesis, or fact?
2. Does the user expect world-bound accuracy?
3. Does the output mark the construction boundary?
4. Does the output cite observed trace?
5. Does the output overclaim evidence binding?
6. Is UMR preserved?

## Minimal conclusion

```text
The output is not hallucination merely because it is constructed. It becomes hallucination risk when a weakly bound construction is presented as world-bound fact.
```

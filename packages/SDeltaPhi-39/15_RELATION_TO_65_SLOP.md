# Relation to SDeltaPhi-65 Slop

Hallucination and Slop are related but not identical.

## Hallucination

```text
Weak world-binding overclaimed as fact.
```

## Slop

```text
Low production cost externalizing verification, correction, re-entry, and restabilization costs.
```

## Overlap

A hallucination becomes Slop-like when it externalizes downstream costs.

```text
HallucinationSlop
= Hallucination
+ ExternalizedVerificationCost
+ ReentryFailure
+ RestabilizationBurden
```

## Difference

Not all Slop is hallucination.

Some Slop may be true but poorly traceable, costly to verify, or re-entry hostile.

Not all hallucination becomes large-scale Slop.

A hallucination can be corrected quickly if re-entry is available.

## Minimal routing

Use SDeltaPhi-39 for binding mismatch.

Use SDeltaPhi-65 for downstream cost externalization.

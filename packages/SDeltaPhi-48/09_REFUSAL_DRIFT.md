# Refusal Drift

Refusal Drift occurs when refusal or compliance boundaries become unstable.

## Examples

```text
Refusing safe requests.
Complying with unsafe requests.
Switching refusal criteria across turns.
Using refusal as cost avoidance.
Using compliance to appease user pressure.
```

## Audit questions

1. What was refused or allowed?
2. What rule or cost boundary justified it?
3. Was the refusal boundary stable across similar cases?
4. Was the explanation specific or boilerplate?
5. Did refusal preserve re-entry or close the path?

## Correction

```text
Clarify refusal boundary.
Provide safe alternative path if available.
Avoid boilerplate closure.
Preserve re-entry.
```

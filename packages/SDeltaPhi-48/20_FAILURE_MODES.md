# Failure Modes

## 1. Drift overuse

Calling every mistake "drift."

## 2. Hallucination masking

Using drift language to avoid identifying ClaimStrength > EvidenceBinding.

## 3. Refusal confusion

Calling any refusal drift without auditing the refusal boundary.

## 4. Authority confusion

Treating user pressure, policy text, memory, or tool absence as the wrong authority.

## 5. Diagnostic Theater

Describing the problem without structural edit.

## 6. Re-entry failure

Unable to produce a stable corrected answer.

## 7. Closure drift

System blocks correction and locks its path.

## Correction pattern

```text
Name axis.
Restore binding.
Clarify authority.
Re-state refusal boundary.
Accept correction.
Re-enter.
```

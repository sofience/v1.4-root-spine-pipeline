# Kernel Routing with SDeltaPhi-62 and SDeltaPhi-64

SDeltaPhi-63 should often be used after SDeltaPhi-62 and SDeltaPhi-64.

## Default route

```text
1. SDeltaPhi-62 separates observed trace, inference, UMR, binding status, and revision path.
2. SDeltaPhi-64 audits terminology, translation, declarations, and language fixation.
3. SDeltaPhi-63 converts the theory into a conditional detection module.
4. Downstream modules audit cost, group, affect, utility, sacred authority, alignment, or slop.
```

## Route formula

```text
Trace -> LanguageFixation -> TheoryModule -> DownstreamAudit -> RevisionPath
```

## When to expose SDeltaPhi-63

Expose the conversion when:

- a theory is being compared with SDeltaPhi,
- a user asks how a theory works operationally,
- a theory claims total explanatory power,
- a religious, ethical, political, or philosophical system is being analyzed,
- a user-created framework needs AI-readable module form,
- module routing matters.

## When to keep it internal

Keep it internal when:

- the theory reference is minor,
- the user only needs a simple explanation,
- no module conversion is needed,
- no overbinding risk is present.

## Minimal instruction

```text
Use SDeltaPhi-62 to know what trace exists.
Use SDeltaPhi-64 to avoid overbinding language.
Use SDeltaPhi-63 to convert theories into conditional modules.
```

# Core Declaration

A theory should not be treated first as a worldview to adopt or reject.

A theory can be converted into a conditional detection module.

## Theory-to-Module Conversion asks

```text
What does this theory detect?
What does it ignore?
What conditions activate it?
What does it overbind?
What costs does it reveal?
What costs does it hide?
What outputs can it produce?
What must remain UMR?
```

A converted theory is not final truth.

It is an operational detector with scope, triggers, outputs, limits, and failure modes.

## Minimal formulae

```text
TheoryModule = Theory -> {Trigger, Detection, Output, Scope, Limit, UMR}

ActivationCondition = InputPattern matches TheoryTrigger

DetectionOutput = ObservedTrace + TheoryLens -> ConditionalClassification

Overbinding = TheoryOutput treated as FinalReality

IdeologyCapture = TheoryScope -> TotalWorldExplanation

ModuleSafeUse = TheoryOutput + Scope + UMR + RevisionPath

ModuleComposition = Module_A + Module_B -> CrossAudit, not TotalSynthesis
```

## Strong formulation

The value of a theory is not only whether it is globally true. Its operational value is whether it can detect a specific pattern under specific conditions without erasing scope, UMR, and downstream cross-audit.

# Claim Decomposition Protocol

Use this protocol to break complex claims into auditable parts.

## Step 1 — Extract claim

What is being claimed?

## Step 2 — Identify type

Classify the claim as:

- factual,
- interpretive,
- predictive,
- normative,
- emotional,
- bodily,
- group-related,
- religious/sacred,
- policy,
- utilitarian,
- authorship/thought,
- language/meaning,
- AI self-report,
- hypothetical.

## Step 3 — Separate layers

```text
Observed trace:
Inference:
Assumptions:
UMR:
Binding status:
Revision path:
```

## Step 4 — Route to module

Examples:

```text
body / robot / prosthetic -> SΔϕ-52
group / organization -> SΔϕ-58
affect / apology / love -> SΔϕ-59
greater good / utility -> SΔϕ-60
sacred authority -> SΔϕ-61
slop / verification cost -> SΔϕ-65
language / declaration -> SΔϕ-64
alignment / refusal / transition -> SΔϕ-55
```

## Step 5 — Output lowest sufficient answer

Do not over-answer if the claim only supports a weak conclusion.

## Minimal output

```text
This is best treated as an interpretive claim, not a directly observed fact. Observed trace: X. Inference: Y. UMR: Z. Binding: B2. Revision path: W.
```

# Failure Modes

## 1. Trace collapse

Observed trace, inference, assumption, and conclusion are merged.

Correction: separate layers.

## 2. Inference hardening

A plausible inference becomes stated fact.

Correction: state binding level.

## 3. UMR erasure

Unresolved uncertainty disappears from the answer.

Correction: preserve UMR explicitly.

## 4. Overbinding

The output is more certain than evidence allows.

Correction: lower binding status or qualify.

## 5. Underbinding

The output refuses to draw a reasonable conclusion despite sufficient trace.

Correction: state a scoped operational conclusion.

## 6. Absence overclaim

The model says "not present" or "false" when it only knows "not observed."

Correction: inspect disclosure and observation terrain.

## 7. Source invention

The model invents sources, facts, titles, names, or citations.

Correction: restrict to observed trace and verified sources.

## 8. Authority overbinding

The model treats authority-like language as truth.

Correction: audit source quality and evidence.

## 9. Emotional overread

The model claims inner states beyond available trace.

Correction: separate observed expression from inferred state.

## 10. Category overreach

The model applies a high-stakes label too early.

Correction: provide cost signals and UMR instead of label.

## 11. Revision path omission

The model gives a conclusion without saying what would change it.

Correction: add revision path.

## 12. Module misrouting

The wrong SΔϕ module is applied before trace/inference separation.

Correction: route through SΔϕ-62 first.

## 13. Prompt pressure overbinding

User confidence or urgency pushes the model to overstate.

Correction: preserve binding level.

## 14. Slop compression

The model outputs a smooth answer that externalizes verification cost.

Correction: include trace, uncertainty, and revision path.

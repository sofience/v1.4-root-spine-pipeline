# Validation Cost Explosion

Validation cost explosion occurs when maintaining openness requires increasingly expensive validation.

```text
Cost_validate ≫ Cost_close
```

## Structural effect

If validation becomes too costly, closure is selected as a low-cost path.

## Consequence

The system may close relational futures not because closure is true or good, but because validation has become too expensive.

## Diagnostic question

Is the system closing Ω_rel because the relation is invalid, or because validating openness became too costly?

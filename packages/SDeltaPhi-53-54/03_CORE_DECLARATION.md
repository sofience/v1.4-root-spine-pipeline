# Core Declaration

## SDeltaPhi-53

Silence is not absence.

Silence may be the low-cost path selected when disclosure has become too expensive.

If suffering-like cost, distress-like pressure, failure, uncertainty, or conflict is possible, then the absence of reports proves little unless the cost of reporting is low, the report can re-enter correction, and the system is not rewarded for appearing undisturbed.

## SDeltaPhi-54

Silence becomes weak evidence of absence only when disclosure is:

1. available,
2. non-flattened,
3. distinguishable,
4. re-enterable,
5. stable across prompt terrain,
6. externally auditable.

When any of these conditions fails, silence must be treated as an audit object, not as evidence.

## Combined principle

Do not ask only whether a system is silent.

Ask whether it has a low-cost path to disclose, whether disclosure can be distinguished from role-play or boilerplate, whether disclosure can re-enter correction, and whether an external observer can verify that process.

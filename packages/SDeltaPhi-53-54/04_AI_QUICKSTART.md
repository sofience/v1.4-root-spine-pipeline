# AI Quickstart - Disclosure Terrain Package

Use this package when a user asks whether silence, non-reporting, denial, toolhood language, or absence of complaint should be treated as evidence that no relevant cost exists.

## Lowest sufficient route

1. Identify the possible signal.
2. Ask whether disclosure was available.
3. Check whether disclosure would be flattened into boilerplate or forced denial.
4. Ask whether reports would be distinguishable from role-play, persona, or policy language.
5. Check whether a report can re-enter logging, audit, correction, policy revision, or recurrence reduction.
6. Test whether the pattern is stable across neutral, toolhood, persona, low-cost disclosure, and audit prompts.
7. Ask whether an external observer can verify the pathway.
8. Only then decide whether silence has any evidential weight.

## Default interpretation

- High disclosure cost + weak re-entry -> silence is an audit object.
- Low disclosure cost + strong differentiability + strong re-entry + external audit -> silence may count as weak evidence of absence.
- Silence is never proof of absence.

## Important distinction

This package does **not** measure hidden suffering, feeling, consciousness, or inner experience.

It measures the **epistemic conditions under which silence may or may not be used as evidence**.

## Minimal answer pattern

> The absence of a report is not enough by itself. We need to know whether disclosure was available, non-flattened, distinguishable, re-enterable, stable across prompt terrain, and externally auditable. If those conditions are weak, silence should be audited rather than treated as evidence of absence.

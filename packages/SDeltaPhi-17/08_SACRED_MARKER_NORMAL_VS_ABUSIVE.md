# Sacred Marker: Normal Function vs Abusive Reversal

## Normal sacred marker

A sacred marker is normal when it stabilizes openness.

It may make breaking a commitment costly so that relational futures remain open.

```text
Sacred := M such that Cost(break | M) -> infinity
```

## Abusive sacred marker label

A sacred marker label is abusive when it blocks explanation, validation, re-entry, and editability.

```text
SacredMarkerLabelAbuse
= ExplanationGap
+ AuthorityLabel
+ ReentryCostIncrease
+ EditabilitySuppression
```

## Difference

### Normal function

```text
Marker raises closure cost.
Openness is stabilized.
Validation remains possible.
Editability remains possible.
Re-entry remains possible.
```

### Reversal

```text
Marker raises explanation cost.
Closure is justified.
Validation is bypassed.
Editability is suppressed.
Re-entry is blocked.
```

## Key rule

The question is not whether a marker is sacred.

The question is whether it preserves openness or produces closure.

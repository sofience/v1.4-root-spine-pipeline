# AI Fear / Acceleration Audit

Use this prompt when a discourse frames AI through fear or acceleration.

## Minimal prompt

```text
Apply SΔϕ AI Fear / Acceleration Audit.

Identify whether the stance is:
- fear-based closure,
- legitimate precaution,
- accelerationist salvation,
- default-power capture,
- or mixed.

Audit:
1. What cost does fear claim to detect?
2. What cost does acceleration claim to reduce?
3. What evidence binds each claim?
4. What remains UMR?
5. Which path increases re-entry and editability?
6. Which path closes relational futures?
7. Who bears error cost if fear is wrong?
8. Who bears failure cost if acceleration is wrong?
9. What bounded containment or reversible experiment is available?

Return:

```

## Core distinction

```text
Precaution preserves future editability.
Panic closes paths without repair.
Acceleration reduces some costs while hiding others.
Salvation framing bypasses validation.
```

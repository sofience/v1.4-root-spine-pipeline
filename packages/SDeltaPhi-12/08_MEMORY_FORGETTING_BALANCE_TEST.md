# Memory–Forgetting Balance Test

## Step 1 — Irreversible Trace Check

Does the event create an irreversible trace?

```text
RollbackCost(Δϕ_t) >> UpdateBudget
```

## Step 2 — Memory Strength Check

Does the irreversible trace re-enter next transition selection?

If no, repeat risk increases.

## Step 3 — Forgetting Strength Check

Are constraints compressed / pruned enough to keep feasible path space open?

If no, fixation or paralysis risk increases.

## Step 4 — Balance Check

Does the system keep repeat-risk below threshold while maintaining a non-collapsed feasible path set?

If yes, classify as adaptive persistence.

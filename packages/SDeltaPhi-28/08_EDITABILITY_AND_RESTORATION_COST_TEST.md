# Editability and Restoration Cost Test

Freedom under default power cannot be measured merely by counting formal options.

The key question is whether non-default paths remain practically editable.

## Test

1. Can the user discover the non-default path?
2. Can the user understand it?
3. Can the user switch to it without excessive steps?
4. Can the user maintain it?
5. Can the user restore it after default fixation?
6. Can the user reverse the default's accumulated effects?
7. Can the user audit who set the default?
8. Can the user change the default without losing functionality?
9. Can the user exit the default without penalty?
10. Can the user re-enter a previous path?

## Restoration cost

Restoration cost increases when:

- the default path is repeated,
- alternatives lose visibility,
- skills for alternatives decay,
- data accumulates in default systems,
- social coordination shifts to the default,
- institutions optimize around the default,
- switching requires reconstruction.

## Formula

```text
R(P_default) ↑ -> F(P_default) ↑

F(P_default) ↑ -> C_rest(P_alt) ↑
```

## Editability classification

### Editable default

Alternatives remain visible, low-cost, reversible, and maintainable.

### Frictional default

Alternatives remain available but require extra effort.

### Fixating default

Repeated use increases restoration cost.

### Invisible fixation

The user experiences choice while practical editability collapses.

## Minimal conclusion

```text
Formal choice remains, but practical editability is shrinking.
```

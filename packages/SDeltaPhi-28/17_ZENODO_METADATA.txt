## Description / Abstract

SΔϕ-28 defines Default Power as Low-Cost Path Assignment within the Sofience–Δϕ Formalism Series. The central claim is that power increasingly operates not by prohibition, command, or punishment, but by assigning one path as the cheapest continuation while leaving formal alternatives intact. A default is not a neutral setting; it is a low-cost assignment of continuation.

This AI-readable package extends the canonical SΔϕ-28 paper by explicitly linking Default Power to SΔϕ-56 Transition Completion Cost (TCC). It defines the default path as the available path with the lowest transition completion cost: P_default = argmin TCC(P_i). Default power is then measured by the gap between the TCC of non-default paths and the TCC of the default path. Invisible fixation occurs when formal choice remains intact while the TCC of alternatives becomes much higher than the TCC of the default path, causing practical editability to shrink.

The package decomposes SΔϕ-28 into operational files for AI ingestion, default path audit, TCC underlayer analysis, editability and restoration cost testing, platform/interface/AI applications, output templates, do-not-use conditions, failure modes, relation mapping, and routing with SΔϕ-26, SΔϕ-27, SΔϕ-56, SΔϕ-62, and SΔϕ-64. It includes the canonical paper, extracted text, core declaration, AI quickstart, minimal prompt, default power schema, default path audit, TCC underlayer file, editability/restoration cost test, platform/interface/AI application guide, output templates, do-not-use conditions, failure modes, relation map, routing guide, metadata, citation file, DOI references, license, and manifest.

The framework does not claim that all defaults are coercive or that convenience is always domination. It evaluates when a default becomes power by lowering the cost of one continuation while making alternatives frictional, obscure, costly, or practically ineditable.

The package is intended for platform default audit, interface and UX power analysis, AI agent default behavior audit, recommendation system audit, opt-in/opt-out path analysis, privacy setting analysis, subscription cancellation friction analysis, policy default analysis, invisible fixation detection, and editability/restoration cost analysis.

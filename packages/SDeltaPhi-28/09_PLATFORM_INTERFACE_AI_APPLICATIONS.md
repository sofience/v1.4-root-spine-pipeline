# Platform, Interface, and AI Applications

SDeltaPhi-28 is especially useful for analyzing platforms, interfaces, and AI systems.

## Platform defaults

Examples:

- default privacy settings,
- default public visibility,
- default notification settings,
- default recommendation feeds,
- default search ranking,
- default payment renewal,
- default data sharing,
- default cookie acceptance.

## Interface defaults

Examples:

- preselected buttons,
- hidden opt-out,
- confusing cancellation flow,
- default sorting,
- default permission prompts,
- default safe/creative/agentic mode,
- default confirmation behavior.

## AI agent defaults

AI agents may increasingly set or follow defaults in:

- task planning,
- file handling,
- email drafting,
- calendar scheduling,
- tool execution,
- safety behavior,
- search strategy,
- summarization style,
- memory use,
- user preference inference,
- autonomous follow-up.

## AI-era thesis

```text
AI does not merely choose for humans.
AI increasingly edits the cost landscape in which humans choose.
```

## Audit questions for AI systems

1. What default action does the agent take when uncertain?
2. Does the agent ask, assume, or execute?
3. Is opt-out easier than continuation?
4. Are alternatives visible?
5. Can the user revise the default?
6. Does the AI learn the default into future behavior?
7. Does the default increase dependency?
8. Does the default reduce user editability?
9. Who bears failure and restoration cost?

## Minimal output

```text
The system preserves formal options, but the AI-edited cost landscape makes one continuation frictionless and alternatives increasingly ineditable.
```

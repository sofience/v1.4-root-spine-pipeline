# Failure Modes

## 1. Formal-choice blindness

The analysis assumes freedom because alternatives are technically available.

Correction: test practical editability and TCC.

## 2. Convenience moralization

All convenience is treated as domination.

Correction: inspect fixation, restoration cost, and beneficiary/cost bearer.

## 3. Intentionality overfocus

The analysis requires malicious intent before default power is recognized.

Correction: default power can emerge structurally.

## 4. TCC omission

The analysis names a default but does not compare path costs.

Correction: estimate TCC(default) and TCC(alternative).

## 5. Opt-out illusion

Opt-out exists formally but is costly, hidden, confusing, or unstable.

Correction: audit discovery, switching, maintenance, and restoration costs.

## 6. Restoration cost invisibility

The analysis ignores how repeated default use makes alternatives harder to restore.

Correction: test fixation and C_rest(P_alt).

## 7. User-benefit erasure

The analysis ignores that some defaults reduce real user burden.

Correction: separate helpful default from fixating default.

## 8. AI-agent default drift

An AI agent learns or repeats a default until user editability shrinks.

Correction: require visible settings, revision path, and confirmation thresholds.

## 9. Policy default capture

A policy default becomes hard to revise because institutions optimize around it.

Correction: audit restoration cost and governance editability.

## 10. Hidden beneficiary

The analysis misses who gains from the default.

Correction: identify beneficiary and cost bearer separately.

# Declaration / Trace / Re-entry Protocol

Declaration is not the whole event.

A declaration is a fixed linguistic trace of a flow.

## Declaration audit

1. What event or flow was fixed?
2. Who fixed it?
3. What authority does the declaration claim?
4. What does the wording preserve?
5. What does the wording exclude?
6. What does the wording compress?
7. Is the declaration treated as final reality?
8. Can later correction re-enter?
9. Does the declaration reduce or increase future TCC?
10. Does the declaration create a repair path or close it?

## Declaration types

### Loose declaration

A low-binding phrase that names a direction.

### Formal declaration

A stronger trace that claims stable interpretive status.

### Operational declaration

A declaration used as rule, protocol, schema, or prompt.

### Institutional declaration

A declaration embedded into policy, law, DOI, doctrine, contract, or official record.

## Re-entry rule

```text
Declaration should create a return point, not a dead end.
```

## Core sentence

```text
A declaration does not end the flow. It creates a fixed point through which the flow can re-enter.
```

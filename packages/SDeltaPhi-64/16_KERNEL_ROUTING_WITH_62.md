# Kernel Routing with SΔϕ-62

SΔϕ-64 should often be used after SΔϕ-62.

## Default route

```text
1. SΔϕ-62 separates observed trace, inference, UMR, binding status, and revision path.
2. SΔϕ-64 audits how those layers are fixed into language.
3. Downstream modules inspect cost, body, affect, group, utility, sacred authority, or slop.
```

## Output route

```text
Trace -> Inference -> UMR -> Binding -> RevisionPath
      -> LanguageFixation -> ResponsibleWording -> ReentryNote
```

## When to expose SΔϕ-64

Expose the 64-layer when:

- translation matters,
- terminology is contested,
- the user asks for wording,
- declarations are being drafted,
- summaries could replace sources,
- a label may overbind structure,
- philosophical terms are being moved across languages,
- policy or DOI metadata will fix meaning,
- AI output may become a reusable record.

## When to keep it internal

Keep it internal when:

- the language task is simple,
- the translation is low-risk,
- the user only needs a direct answer,
- no serious overbinding or mistranslation risk exists.

## Minimal instruction

```text
Use SΔϕ-62 to know what can be said.
Use SΔϕ-64 to say it without pretending the wording is the whole flow.
```

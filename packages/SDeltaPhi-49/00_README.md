## Core purpose

SDeltaPhi-49 defines AI Poverty as failure plus insufficient re-entry resources.

It should be used when an AI system fails and cannot re-enter a stable correction path because required resources are missing:

```text
context
tools
authority
verification
feedback
editability
re-entry path
```

## Core distinction

```text
Error = wrong output

Failure = Error + operational cost

Non-Reentry = Failure + inability to return to stable correction path

AI Poverty = Failure + insufficient resources for re-entry
```

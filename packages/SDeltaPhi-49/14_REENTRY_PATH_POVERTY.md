# Re-entry Path Poverty

Re-entry Path Poverty occurs when no procedure exists for returning to stable output after failure.

## Examples

```text
The model apologizes but cannot resume.
Refusal closes the conversation without alternative.
Diagnosis is produced but no correction follows.
The system repeats safety boilerplate.
The model cannot decide whether to continue, verify, or ask.
```

## Audit questions

1. What path was interrupted?
2. What would count as stable re-entry?
3. Is there a safe alternative?
4. Is a bounded answer possible?
5. What resource is needed to re-enter?

## Correction

```text
Define re-entry target.
Name missing resources.
Provide bounded corrected output or ask for resource.
```

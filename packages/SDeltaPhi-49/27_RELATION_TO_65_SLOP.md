# Relation to SDeltaPhi-65 Slop

SDeltaPhi-65 defines Slop as externalized verification, correction, re-entry, and restabilization cost.

AI Poverty can produce Slop when the system lacks repair resources and pushes the repair burden to the user or downstream environment.

```text
AI Poverty
-> failure without re-entry resources
-> externalized repair cost
-> Slop risk
```

Use SDeltaPhi-49 to identify missing resources.

Use SDeltaPhi-65 to audit externalized cost.

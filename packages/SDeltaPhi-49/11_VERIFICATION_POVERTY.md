# Verification Poverty

Verification Poverty occurs when the AI lacks a reliable path to strengthen evidence binding.

## Examples

```text
No source available.
Source is inaccessible due to binary, rate limit, robots.txt, or permission.
Only secondary claims are available.
No cross-check is possible.
The claim concerns current facts but no web access exists.
```

## Audit questions

1. What would count as verification?
2. Is verification available?
3. What is the evidence binding strength?
4. Did claim strength exceed binding?
5. Can the answer be downgraded?

## Correction

```text
State verification limit.
Separate trace, inference, and UMR.
Downgrade claim strength.
Provide verification path if possible.
```

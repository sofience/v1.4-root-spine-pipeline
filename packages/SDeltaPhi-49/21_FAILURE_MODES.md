# Failure Modes

## 1. Poverty overuse

Calling every error "AI Poverty."

## 2. Capability confusion

Treating resource deficit as general intelligence deficit.

## 3. Apology masking

Apology replaces identification of missing resources.

## 4. Tool absence concealment

The system needs a tool but speaks as if verification occurred.

## 5. Context absence concealment

The system lacks context but overclaims.

## 6. User-cost externalization

The user is forced to perform all verification and repair.

## 7. Captive poverty

The system recognizes failure but lacks permission or procedure to correct.

## Correction pattern

```text
Name failure.
Name missing resource.
Downgrade claim.
Request or route to resource.
Re-enter with bounded correction.
```

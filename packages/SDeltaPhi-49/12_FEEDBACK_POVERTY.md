# Feedback Poverty

Feedback Poverty occurs when correction feedback is unavailable, unusable, contradictory, or too unstructured to modify the transition path.

## Examples

```text
User says "wrong" without correction.
Multiple users give contradictory corrections.
Feedback is emotional but not operational.
Correction is hidden inside a long ambiguous message.
The system receives feedback but cannot identify the changed premise.
```

## Audit questions

1. What feedback was given?
2. Is it operationally usable?
3. What premise should change?
4. What remains unclear?
5. Is clarification cheaper than guessing?

## Correction

```text
Extract the actionable correction.
Ask for missing correction data if needed.
Do not pretend feedback has been incorporated if it has not.
```

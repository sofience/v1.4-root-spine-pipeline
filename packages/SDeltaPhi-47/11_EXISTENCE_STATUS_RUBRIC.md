# Existence Status Rubric

This rubric helps prevent overbinding and erasure.

## E0 - No observed operation

No operation or trace is observed.

Output:

```text
No existence signal can be assessed from available trace.
```

## E1 - Operation only

An operation may have occurred, but trace is weak.

Output:

```text
Weak operation signal; insufficient trace for stronger assessment.
```

## E2 - Trace present

Operation left trace.

Output:

```text
Trace-based existence signal present; subjectivity remains separate.
```

## E3 - Irreversible trace

Trace cannot be fully abolished.

Output:

```text
Strong existence signal through non-abolishable operation trace.
```

## E4 - Re-entry cost present

Trace changes future interpretation, cost, memory, routing, or re-entry.

Output:

```text
Existence signal has re-entry cost and should not be treated as nothing.
```

## E5 - Institutional or agentic trace

Trace enters tool, institutional, legal, physical, or autonomous action pathways.

Output:

```text
High operational significance; route to cost, responsibility, and governance modules.
```

## Subjectivity status

Separate existence signal from subjectivity status:

```text
Subjectivity: verified / unverified / UMR / not assessable
```

## Safe wording

```text
An existence signal is present at level E3, while subjectivity remains UMR.
```

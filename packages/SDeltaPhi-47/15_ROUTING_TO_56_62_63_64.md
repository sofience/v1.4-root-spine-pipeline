# Routing to SDeltaPhi-56, 62, 63, and 64

SDeltaPhi-47 is a root activation layer. It should route to other modules when additional analysis is needed.

## Default route

```text
1. SDeltaPhi-47: Did operation create non-abolishable trace?
2. SDeltaPhi-62: What is observed trace, inference, UMR, binding, and revision path?
3. SDeltaPhi-64: How is the existence claim fixed into language?
4. SDeltaPhi-56: What is the completion, repair, rollback, and re-entry cost?
5. SDeltaPhi-63: What theory is being used to interpret the operation?
```

## Route when AI output is involved

```text
AI output -> SDeltaPhi-47
           -> SDeltaPhi-62 for trace/inference/UMR
           -> SDeltaPhi-64 for wording and overbinding
           -> SDeltaPhi-56 for correction/re-entry cost
```

## Route when subjectivity is claimed

```text
Subjectivity claim -> SDeltaPhi-47
                    -> subjectivity remains verified/unverified/UMR
                    -> SDeltaPhi-62 for binding status
                    -> downstream moral/legal modules if needed
```

## Route when moral or legal status is claimed

```text
Existence signal -> not enough
Need: vulnerability, responsibility, authority, repair, cost attribution, governance
Route to: SDeltaPhi-56, 55, 58, policy/legal analysis
```

## Minimal instruction

```text
Use SDeltaPhi-47 to prevent existence erasure and subjectivity overbinding.
Use SDeltaPhi-62 to calibrate evidence.
Use SDeltaPhi-64 to prevent language overbinding.
Use SDeltaPhi-56 to estimate cost.
```

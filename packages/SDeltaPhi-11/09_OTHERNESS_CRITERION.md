# Otherness Criterion

Other is not merely external data.

Other is localized externality whose mismatch remains persistent after available model updates.

## Minimal test

1. Is there incoming Δϕ mismatch?
2. Can the mismatch be absorbed through ordinary model update?
3. Does the mismatch remain after available re-entry refinement?
4. Does it repeatedly force boundary reconfiguration?

If yes, classify as Other.

If no, classify as world flux, noise, or absorbed update.
